1. Introducing Embarcadero KAI

KAI is Embarcadero's IDE-aware assistant for RAD Studio. It works directly with the active Delphi environment: the open project, editor buffers, unsaved changes, project files, design-time packages and compiler output.

In this project, KAI was used as a development assistant to transform a simple notification procedure into an installable Delphi component with embedded resources, design-time property editors, palette icons and a written implementation report.

Main Strengths

  • IDE-first workflow: KAI reads RAD Studio editor buffers before reading files directly from disk.
  • Protection of unsaved changes: edits are applied to the IDE buffer when the file is open.
  • Active project awareness: KAI can inspect the package, platform, configuration and project file list.
  • Targeted source edits: changes can be applied precisely with applyEdit, without rewriting unrelated code.
  • Project management: KAI can add units and resource files to the active RAD Studio project.
  • Continuous validation: KAI can compile the project through RAD Studio after each important change.
  • Context-aware Delphi support: the implementation was adapted to VCL, design-time packages, BPL resources and WinRT toast APIs.
  • Traceability: the workflow can be documented as a reproducible report.

4. Component Creation Steps

The following sections describe the technical implementation steps: project analysis, component creation, Windows toast handling, embedded resources, design-time support, validation and maintenance.

4.1 Initial Project Analysis

KAI first inspected the existing package. The project contained mainly notify_Unit.pas, modern_notification.dpk, modern_notification.dproj and related resource files.

The initial unit contained a global ShowNotification procedure using WinRT toast APIs: TToastNotificationManager, IToastNotification and Windows toast XML templates. It was not yet exposed as an installable component.

KAI compiled the project to establish a valid starting point. The baseline build succeeded, which allowed the component to be expanded step by step with validation after each important change.

4.2 Transformation into a Delphi Component

The runtime code was reorganized around a non-visual component:

type
  TModernNotification = class(TComponent)
  published
    property AppID: string;
    property Body: string;
    property DefaultIcon: Boolean;
    property DefaultIconName: string;
    property IconFileName: string;
    property Title: string;
    property UseLargeIcon: Boolean;
    property Version: string;
  public
    procedure Show; overload;
    procedure Show(const ATitle, ABody: string); overload;
  end;

The global ShowNotification helper was kept so that a notification can still be displayed without dropping the component on a form.

4.3 Windows Notification Display

The component builds a toast XML document from Windows templates:

  • ToastImageAndText02 when an icon is available.
  • ToastText02 when no icon should be displayed.

Text values are written into the XML text nodes. When an image is used, it is converted to a local file:///... URI and assigned to the XML image node.

The UseLargeIcon property adds:

placement="appLogoOverride"
hint-crop="none"

This produces a large Windows 10/11 style notification icon.

4.4 Default Icon Embedded in the BPL

The default icon was embedded through modern_notification_resources.rc, then compiled into modern_notification_resources.res.

Because toast notifications need a file URI for local images, the embedded PNG is extracted to the Windows temporary folder before being used. 

LModuleHandle := FindClassHInstance(TModernNotification);
LResourceStream := TResourceStream.Create(LModuleHandle, ResourceName, RT_RCDATA);

Using FindClassHInstance ensures that the resource is loaded from the BPL containing the component.

4.5 External Icon or Embedded Icon

The final resolution order is:

  1. If IconFileName points to an existing file, that file is used.
  2. If DefaultIcon=False, no icon is used.
  3. Otherwise the resource selected by DefaultIconName is extracted and used.
  4. If the name is empty or invalid, the component falls back to button-info.

4.7 Design-Time Unit

A separate notify_Design.pas unit was created to isolate code that depends on designide, DesignIntf, DesignEditors and ToolsAPI.

It registers the component:

RegisterComponents('Modern Notification', [TModernNotification]);

It also registers property editors:

  • IconFileName: ... button opening a TOpenDialog.
  • DefaultIconName: drop-down list of embedded icons.
  • Version: read-only property.

4.8 Component Icon and RAD Studio Splash

The design-time resource file contains:

TMODERNNOTIFICATION BITMAP "images\\flag-green.bmp"
TMODERNNOTIFICATION_32 BITMAP "images\\flag-green_32.bmp"

TMODERNNOTIFICATION is the resource name RAD Studio expects for the component palette icon. TMODERNNOTIFICATION_32 is used for the splash/plugin bitmap. 

{$R 'modern_notification_design.res'}

4.9 Code Cleanup

A cleanup pass was performed after the main additions. Functions still present are used directly or indirectly by published properties, runtime behavior or design-time initialization.

Unused units removed:

4.11 Final Component Usage

Using an embedded icon:

ModernNotification1.Title := 'Information';
ModernNotification1.Body := 'Processing completed';
ModernNotification1.DefaultIcon := True;
ModernNotification1.DefaultIconName := 'button-info';
ModernNotification1.Show;

Using an external icon:

ModernNotification1.Title := 'Alert';
ModernNotification1.Body := 'Operation completed with an error';
ModernNotification1.IconFileName := 'C:\Images\error_64.png';
ModernNotification1.Show;

Quick use without dropping the component on a form:

ShowNotification('Information', 'Processing completed');

4.13 Maintenance Notes

  • After changing a design-time resource, restart RAD Studio or reinstall the package to see the new palette icon.
  • Windows toast notifications use file URIs, so embedded PNG resources are extracted before display.
  • To add another embedded icon, add a 64x64 PNG to images, declare it in the .rc file and add it to EmbeddedIcons.
  • Continue using KAI for files open in the IDE to avoid overwriting unsaved changes.
